# Phase 5.5: Entity Cache

## Overview

The Entity Cache provides an in-memory index of all entities in the system, enabling O(1) path resolution and model dispatch without database queries. This is the bridge between the VFS (path-based) and the model system (UUID-based).

## Problem Statement

When VFS receives a request for `/home/user/docs/report.pdf`:
1. It must resolve the path to an entity UUID
2. It must know which model handles that entity (file? video? audio?)
3. It must dispatch to the correct model's table for metadata

Without a cache, each path component requires a database query. With 1 million files, this becomes a bottleneck.

## Architecture

### Central Entity Registry

Every entity in the system is registered in a lightweight in-memory cache:

```typescript
interface CachedEntity {
    id: string;        // Entity UUID
    model: string;     // Model name (file, folder, video, etc.)
    parent: string | null;  // Parent entity UUID
    name: string;      // Entity name (filename)
}
```

**Memory footprint per entity:** ~200-250 bytes
**1 million entities:** ~250-300 MB

### Cache Structure

```typescript
interface EntityCache {
    /**
     * Primary lookup by UUID.
     * O(1) access to any entity's minimal metadata.
     */
    byId: Map<string, CachedEntity>;

    /**
     * Child index for path resolution.
     * Key: "parentId:childName" → Value: childId
     * Enables O(1) path component lookups.
     */
    childIndex: Map<string, string>;

    /**
     * Optional: Children listing for readdir().
     * Key: parentId → Value: Set of child IDs
     * If not maintained, readdir scans byId (still fast at 1M scale).
     */
    childrenOf?: Map<string, Set<string>>;
}
```

### Operations

#### Path Resolution (downward)

```typescript
function resolvePath(path: string): string | null {
    const components = path.split('/').filter(Boolean);
    let currentId = ROOT_ID;

    for (const name of components) {
        const key = `${currentId}:${name}`;
        const childId = cache.childIndex.get(key);
        if (!childId) return null;
        currentId = childId;
    }

    return currentId;
}
```

**Cost:** O(path depth) Map lookups. Zero SQL.

#### Path Computation (upward)

```typescript
function computePath(id: string): string {
    const parts: string[] = [];
    let current = cache.byId.get(id);

    while (current && current.parent) {
        parts.unshift(current.name);
        current = cache.byId.get(current.parent);
    }

    return '/' + parts.join('/');
}
```

**Cost:** O(path depth) Map lookups. Zero SQL.

#### Model Dispatch

```typescript
function getModelForEntity(id: string): string | null {
    const entity = cache.byId.get(id);
    return entity?.model ?? null;
}
```

**Cost:** O(1). Zero SQL.

#### List Children

```typescript
function listChildren(parentId: string): string[] {
    // Option A: Use childrenOf index (O(1))
    const children = cache.childrenOf?.get(parentId);
    if (children) return [...children];

    // Option B: Scan byId (O(n) but still fast)
    const results: string[] = [];
    for (const [id, entity] of cache.byId) {
        if (entity.parent === parentId) {
            results.push(id);
        }
    }
    return results;
}
```

## Cache Lifecycle

### Boot: Initial Load

```typescript
async function loadEntityCache(db: DatabaseConnection): Promise<EntityCache> {
    const cache: EntityCache = {
        byId: new Map(),
        childIndex: new Map(),
        childrenOf: new Map(),
    };

    // Load from ALL model tables
    const models = await db.query<{ model_name: string }>(
        "SELECT model_name FROM models WHERE status = 'system' OR status = 'active'"
    );

    for (const { model_name } of models) {
        // Query minimal fields from each table
        const entities = await db.query<CachedEntity>(
            `SELECT id, '${model_name}' as model, parent, name FROM ${model_name} WHERE trashed_at IS NULL`
        );

        for (const entity of entities) {
            cache.byId.set(entity.id, entity);

            if (entity.parent) {
                cache.childIndex.set(`${entity.parent}:${entity.name}`, entity.id);

                // Optional: maintain childrenOf
                let children = cache.childrenOf.get(entity.parent);
                if (!children) {
                    children = new Set();
                    cache.childrenOf.set(entity.parent, children);
                }
                children.add(entity.id);
            }
        }
    }

    return cache;
}
```

**Cost:** One query per model table. ~200-500ms for 1M entities.

### Mutations: Keep Cache In Sync

The observer pipeline (Ring 8) maintains cache consistency:

```typescript
// In Ring 8 CacheObserver

function onEntityCreated(model: string, entity: { id, parent, name }): void {
    const cached: CachedEntity = {
        id: entity.id,
        model,
        parent: entity.parent,
        name: entity.name,
    };

    cache.byId.set(entity.id, cached);

    if (entity.parent) {
        cache.childIndex.set(`${entity.parent}:${entity.name}`, entity.id);
        cache.childrenOf?.get(entity.parent)?.add(entity.id);
    }
}

function onEntityUpdated(id: string, changes: Partial<CachedEntity>): void {
    const existing = cache.byId.get(id);
    if (!existing) return;

    // Handle rename
    if (changes.name && changes.name !== existing.name) {
        cache.childIndex.delete(`${existing.parent}:${existing.name}`);
        cache.childIndex.set(`${existing.parent}:${changes.name}`, id);
        existing.name = changes.name;
    }

    // Handle move (parent change)
    if (changes.parent !== undefined && changes.parent !== existing.parent) {
        // Remove from old parent
        cache.childIndex.delete(`${existing.parent}:${existing.name}`);
        cache.childrenOf?.get(existing.parent)?.delete(id);

        // Add to new parent
        if (changes.parent) {
            cache.childIndex.set(`${changes.parent}:${existing.name}`, id);
            cache.childrenOf?.get(changes.parent)?.add(id);
        }
        existing.parent = changes.parent;
    }
}

function onEntityDeleted(id: string): void {
    const existing = cache.byId.get(id);
    if (!existing) return;

    cache.byId.delete(id);

    if (existing.parent) {
        cache.childIndex.delete(`${existing.parent}:${existing.name}`);
        cache.childrenOf?.get(existing.parent)?.delete(id);
    }
}
```

## VFS Integration

### Before (without cache)

```typescript
async stat(path: string): Promise<ModelStat> {
    // 1. Resolve path (N database queries for N components)
    const id = await this.resolvePath(path);  // SLOW

    // 2. Figure out which model (another query)
    const entity = await storage.get(`entity:${id}`);  // SLOW
    const model = this.models.get(entity.model);

    // 3. Get full metadata
    return model.stat(ctx, id);
}
```

### After (with cache)

```typescript
async stat(path: string): Promise<ModelStat> {
    // 1. Resolve path (N in-memory lookups)
    const id = this.entityCache.resolvePath(path);  // FAST
    if (!id) throw new ENOENT(path);

    // 2. Get model from cache (1 in-memory lookup)
    const modelName = this.entityCache.getModel(id);  // FAST
    const model = this.models.get(modelName);

    // 3. Get full metadata (only DB query)
    return model.stat(ctx, id);
}
```

## Dynamic Models

When a vendor creates a new model (e.g., `video`):

1. **DDL Observer** creates the `video` table
2. **Cache** doesn't need changes - it just learns about `video` entities as they're created
3. **VFS** routes `stat()` calls to DatabaseOps with `model: 'video'`

The cache doesn't care about model-specific fields. It only tracks the minimal path-resolution fields that all entities share.
