# Phase 3.6: Stream-First Database Layer

## Goal

Refactor DatabaseService to be stream-first internally (kernel space) while providing a userspace wrapper with familiar array-based API.

## Problem

Phase 3.5 DatabaseService uses array-in/array-out patterns (`Promise<T[]>`), which:
- Buffers entire result sets in memory
- Doesn't compose naturally with MessagePipe
- Forces observer pipeline to process batches, not streams

## Solution

Two layers with different responsibilities:

| Layer | Input | Output | Used By |
|-------|-------|--------|---------|
| Kernel (`DatabaseOps`) | `Source<T>` | `AsyncGenerator<T>` | Internal kernel, pipes |
| Userspace (`DatabaseService`) | `T[]` | `Promise<T[]>` | Shell commands, public API |

## Kernel Layer: `DatabaseOps`

The internal streaming interface. All methods accept iterables and yield records.

```typescript
/**
 * Source type - accepts sync or async iterables.
 * Arrays are Iterable<T>, pipes/generators are AsyncIterable<T>.
 */
type Source<T> = Iterable<T> | AsyncIterable<T>;

class DatabaseOps {
    // -------------------------------------------------------------------------
    // Select Operations (bypass observer pipeline)
    // -------------------------------------------------------------------------

    async *selectAny<T>(model: string, filter?: FilterData, options?: SelectOptions): AsyncGenerator<T>;
    async *selectIds<T>(model: string, ids: Source<string>, options?: SelectOptions): AsyncGenerator<T>;

    // -------------------------------------------------------------------------
    // Create Operations (through observer pipeline)
    // -------------------------------------------------------------------------

    async *createAll<T>(model: string, source: Source<CreateInput<T>>): AsyncGenerator<T>;

    // -------------------------------------------------------------------------
    // Update Operations (through observer pipeline)
    // -------------------------------------------------------------------------

    async *updateAll<T>(model: string, source: Source<UpdateInput<T>>): AsyncGenerator<T>;
    async *updateIds<T>(model: string, ids: Source<string>, changes: Partial<T>): AsyncGenerator<T>;
    async *updateAny<T>(model: string, filter: FilterData, changes: Partial<T>): AsyncGenerator<T>;

    // -------------------------------------------------------------------------
    // Delete Operations - Soft Delete (through observer pipeline)
    // -------------------------------------------------------------------------

    async *deleteAll<T>(model: string, source: Source<DeleteInput>): AsyncGenerator<T>;
    async *deleteIds<T>(model: string, ids: Source<string>): AsyncGenerator<T>;
    async *deleteAny<T>(model: string, filter: FilterData): AsyncGenerator<T>;

    // -------------------------------------------------------------------------
    // Revert Operations - Undo Soft Delete (through observer pipeline)
    // -------------------------------------------------------------------------

    async *revertAll<T>(model: string, source: Source<RevertInput>): AsyncGenerator<T>;
    async *revertAny<T>(model: string, filter?: FilterData): AsyncGenerator<T>;

    // -------------------------------------------------------------------------
    // Expire Operations - Hard Delete (through observer pipeline)
    // -------------------------------------------------------------------------

    async *expireAll<T>(model: string, source: Source<DeleteInput>): AsyncGenerator<T>;

    // -------------------------------------------------------------------------
    // Upsert Operations (through observer pipeline)
    // -------------------------------------------------------------------------

    async *upsertAll<T>(model: string, source: Source<CreateInput<T> | UpdateInput<T>>): AsyncGenerator<T>;
}
```

### Key Properties

- **Composable**: Output of one operation can feed into another
- **Backpressure**: Slow consumers naturally slow producers via async iteration
- **Memory efficient**: No buffering - records flow through one at a time
- **Uniform**: Same pattern regardless of input size

## Userspace Layer: `DatabaseService`

Thin wrapper that collects streaming results into arrays. Provides the familiar API for shell commands and userspace programs.

```typescript
class DatabaseService {
    constructor(private ops: DatabaseOps) {}

    // -------------------------------------------------------------------------
    // Select Operations
    // -------------------------------------------------------------------------

    async selectAny<T>(model: string, filter?: FilterData, options?: SelectOptions): Promise<T[]> {
        return collect(this.ops.selectAny<T>(model, filter, options));
    }

    async selectOne<T>(model: string, filter: FilterData, options?: SelectOptions): Promise<T | null> {
        for await (const record of this.ops.selectAny<T>(model, { ...filter, limit: 1 }, options)) {
            return record;
        }
        return null;
    }

    async select404<T>(model: string, filter: FilterData, message?: string): Promise<T> {
        const record = await this.selectOne<T>(model, filter);
        if (!record) throw new Error(message ?? `${model} not found`);
        return record;
    }

    async selectIds<T>(model: string, ids: string[], options?: SelectOptions): Promise<T[]> {
        return collect(this.ops.selectIds<T>(model, ids, options));
    }

    async count(model: string, filter?: FilterData, options?: SelectOptions): Promise<number> {
        let count = 0;
        for await (const _ of this.ops.selectAny(model, filter, options)) {
            count++;
        }
        return count;
    }

    // -------------------------------------------------------------------------
    // Create Operations
    // -------------------------------------------------------------------------

    async createAll<T>(model: string, records: CreateInput<T>[]): Promise<T[]> {
        return collect(this.ops.createAll<T>(model, records));
    }

    async createOne<T>(model: string, record: CreateInput<T>): Promise<T> {
        for await (const created of this.ops.createAll<T>(model, [record])) {
            return created;
        }
        throw new Error('Create failed');
    }

    // -------------------------------------------------------------------------
    // Update Operations
    // -------------------------------------------------------------------------

    async updateAll<T>(model: string, updates: UpdateInput<T>[]): Promise<T[]> {
        return collect(this.ops.updateAll<T>(model, updates));
    }

    async updateOne<T>(model: string, id: string, changes: Partial<T>): Promise<T> {
        for await (const updated of this.ops.updateAll<T>(model, [{ id, changes }])) {
            return updated;
        }
        throw new Error('Update failed');
    }

    // ... etc for delete, revert, expire, upsert
}
```

## Helper Functions

```typescript
/**
 * Collect an async iterable into an array.
 * Used at userspace boundary to materialize streams.
 */
async function collect<T>(source: AsyncIterable<T>): Promise<T[]> {
    const results: T[] = [];
    for await (const item of source) {
        results.push(item);
    }
    return results;
}

/**
 * Normalize sync/async iterables to async.
 * Used internally by DatabaseOps.
 */
async function* normalize<T>(source: Source<T>): AsyncGenerator<T> {
    for await (const item of source) {
        yield item;
    }
}
```

## Observer Pipeline Integration

ObserverRunner becomes pipe-based, processing one record at a time:

```typescript
class ObserverRunner {
    /**
     * Run records through the observer pipeline.
     * Each ring is a stage in the pipe.
     */
    async *run<T>(
        op: OperationType,
        model: Model,
        source: Source<ModelRecord>
    ): AsyncGenerator<ModelRecord> {
        // Build pipeline once: Ring0 | Ring1 | ... | Ring9
        const observers = this.getObservers(op, model);

        for await (const record of normalize(source)) {
            // Flow record through all rings
            let current = record;

            for (const ring of ObserverRing.ordered()) {
                const ringObservers = observers.filter(o => o.ring === ring);

                // Observers at same ring run in parallel
                await Promise.all(
                    ringObservers.map(o => o.execute({ op, model, record: current }))
                );

                // Record may have been modified by observers
            }

            yield current;
        }
    }
}
```

### Backpressure Behavior

- If Ring 5 is slow, it creates backpressure
- Rings 1-4 slow down (waiting to yield)
- Rings 6-9 starve (no input yet)
- Source production slows naturally
- No explicit queue management needed

## Usage Examples

### Kernel-space (pipes)

```typescript
// Pipe records from one model to another
const source = ops.selectAny<FileRecord>('file', { where: { status: 'pending' } });
const created = ops.createAll<ArchiveRecord>('archive', transform(source));

for await (const record of created) {
    // Each record streams through as it completes
}
```

### Userspace (arrays)

```typescript
// Familiar array-based API
const files = await db.selectAny<FileRecord>('file', { where: { owner: userId } });
const created = await db.createAll<FileRecord>('file', newFiles);
```

### Chaining transforms

```typescript
// Compose operations
const pipeline = ops.selectAny('file', filter)
    |> filterAsync(isValid)
    |> mapAsync(transform)
    |> ops.createAll('archive', %);
```
