# Phase 3.5: Database API Alignment

## Goal

Align the DatabaseService API with the original monk-api patterns, including:
- Deliberate naming conventions (`*All`, `*Any`, `*Ids`, `*One`, `*404`)
- Complete operation set (`select`, `create`, `update`, `delete`, `revert`, `expire`, `upsert`)
- Filter system for complex queries
- Streaming support via async generators

## API Naming Conventions

| Suffix | Input Type | Description |
|--------|-----------|-------------|
| `*All()` | Record array | Operates on array of full records |
| `*Any()` | FilterData | Uses filter criteria to find records |
| `*Ids()` | UUID array | Operates on specific record IDs |
| `*One()` | Single record/ID | Single record operation |
| `*404()` | FilterData | Like `*One` but throws if not found |

## Operations

### Select Operations (bypass observer pipeline)

```typescript
// Filter-based selection
selectAny<T>(modelName: string, filter?: FilterData, options?: SelectOptions): Promise<T[]>
selectOne<T>(modelName: string, filter: FilterData, options?: SelectOptions): Promise<T | null>
select404<T>(modelName: string, filter: FilterData, message?: string, options?: SelectOptions): Promise<T>

// ID-based selection
selectIds<T>(modelName: string, ids: string[], options?: SelectOptions): Promise<T[]>

// Record-based refresh (re-select by id from existing records)
selectAll<T>(modelName: string, records: T[]): Promise<T[]>

// Aggregation
count(modelName: string, filter?: FilterData, options?: SelectOptions): Promise<number>

// Streaming
streamAny<T>(modelName: string, filter?: FilterData, options?: SelectOptions): AsyncGenerator<T>
streamIds<T>(modelName: string, ids: string[], options?: SelectOptions): AsyncGenerator<T>
```

### Create Operations (through observer pipeline)

```typescript
createAll<T>(modelName: string, records: CreateInput<T>[]): Promise<T[]>
createOne<T>(modelName: string, record: CreateInput<T>): Promise<T>
```

### Update Operations (through observer pipeline)

```typescript
updateAll<T>(modelName: string, updates: UpdateInput<T>[]): Promise<T[]>
updateOne<T>(modelName: string, id: string, changes: Partial<T>): Promise<T>
updateIds<T>(modelName: string, ids: string[], changes: Partial<T>): Promise<T[]>
updateAny<T>(modelName: string, filter: FilterData, changes: Partial<T>): Promise<T[]>
update404<T>(modelName: string, filter: FilterData, changes: Partial<T>, message?: string): Promise<T>
```

### Delete Operations - Soft Delete (through observer pipeline)

```typescript
deleteAll<T>(modelName: string, deletes: DeleteInput[]): Promise<T[]>
deleteOne<T>(modelName: string, id: string): Promise<T>
deleteIds<T>(modelName: string, ids: string[]): Promise<T[]>
deleteAny<T>(modelName: string, filter: FilterData): Promise<T[]>
delete404<T>(modelName: string, filter: FilterData, message?: string): Promise<T>
```

### Revert Operations - Undo Soft Delete (through observer pipeline)

```typescript
revertAll<T>(modelName: string, reverts: RevertInput[]): Promise<T[]>
revertOne<T>(modelName: string, id: string): Promise<T>
revertAny<T>(modelName: string, filter?: FilterData): Promise<T[]>
```

### Expire Operations - Hard Delete (through observer pipeline)

```typescript
expireAll<T>(modelName: string, expires: DeleteInput[]): Promise<T[]>
expireOne<T>(modelName: string, id: string): Promise<T>
```

### Upsert Operations (through observer pipeline)

```typescript
upsertAll<T>(modelName: string, records: (CreateInput<T> | UpdateInput<T>)[]): Promise<T[]>
upsertOne<T>(modelName: string, record: CreateInput<T> | UpdateInput<T>): Promise<T>
```

## Filter System

### FilterData Structure

```typescript
interface FilterData {
    where?: WhereConditions;
    order?: OrderSpec;
    limit?: number;
    offset?: number;
    select?: string[];
}
```

### FilterOp - Query Operators

```typescript
enum FilterOp {
    // Comparison
    EQ = '$eq',      // Equal (implicit default)
    NE = '$ne',      // Not equal
    GT = '$gt',      // Greater than
    GTE = '$gte',    // Greater than or equal
    LT = '$lt',      // Less than
    LTE = '$lte',    // Less than or equal

    // Pattern matching
    LIKE = '$like',      // SQL LIKE
    ILIKE = '$ilike',    // Case-insensitive LIKE
    NLIKE = '$nlike',    // NOT LIKE
    NILIKE = '$nilike',  // NOT case-insensitive LIKE

    // Array membership
    IN = '$in',      // Value in array
    NIN = '$nin',    // Value not in array

    // Logical
    AND = '$and',    // All conditions must match
    OR = '$or',      // Any condition must match
    NOT = '$not',    // Negate condition

    // Range
    BETWEEN = '$between',  // Value between two values

    // Null checks
    EXISTS = '$exists',    // Field is not null
    NULL = '$null',        // Field is null
}
```

### Filter Examples

```typescript
// Simple equality (implicit $eq)
{ where: { status: 'active' } }

// Explicit operator
{ where: { age: { $gte: 18 } } }

// Pattern matching
{ where: { name: { $ilike: 'john%' } } }

// Array membership
{ where: { status: { $in: ['active', 'pending'] } } }

// Logical operators
{ where: { $and: [{ status: 'active' }, { age: { $gte: 18 } }] } }
{ where: { $or: [{ role: 'admin' }, { role: 'moderator' }] } }

// Complex nested
{
    where: {
        $and: [
            { status: 'active' },
            { $or: [{ role: 'admin' }, { verified: true }] }
        ]
    },
    order: [{ field: 'created_at', sort: 'desc' }],
    limit: 10
}
```

## SelectOptions

```typescript
interface SelectOptions {
    /** Include soft-deleted records */
    trashed?: 'exclude' | 'include' | 'only';
}
```

## Input Types

```typescript
// Create input - record data without system fields
type CreateInput<T> = Omit<T, 'id' | 'created_at' | 'updated_at' | 'trashed_at' | 'expired_at'>;

// Update input - record with id and partial changes
interface UpdateInput<T> {
    id: string;
    changes: Partial<T>;
}

// Delete/Revert input - just the id
interface DeleteInput {
    id: string;
}
```

## Implementation Notes

1. **Filter class**: Port from monk-api with SQLite adaptations
2. **SQL generation**: Use parameterized queries via DatabaseConnection
3. **Streaming**: Use async generators yielding individual records
4. **Observer integration**: All mutations flow through ObserverRunner
